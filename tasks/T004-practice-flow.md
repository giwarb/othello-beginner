## 要件

### 状態機械(UI から独立した純粋ロジックにすること)

1. 状態: `placing`(おく段階) → `flipping`(ひっくりかえす段階) → `result`(結果) と、各状態のデータ(盤面・手番・置いた位置・裏返すべき石の集合・裏返し済み集合・ミスカウント3種)。
2. `placing` でのタップイベント処理:
   - 合法手 → 石を置き `flipping` へ。**盤面はまだ裏返さない**(石を1個追加するだけ。裏返しは子どもの操作で1枚ずつ)
   - 非合法(空きマス・石のあるマス問わず) → メッセージ「そこには　おけないよ！」、おくミス +1
3. `flipping` でのタップイベント処理:
   - 裏返すべき石(`flippedDiscs` の結果に含まれ、未裏返し) → その1枚を裏返す(盤面更新)。全部裏返しても自動で結果に進まない(OK を押させる)
   - それ以外の石・マス → メッセージ「そのこまは　ひっくりかえせないよ！」、ひっくりかえしミス +1
   - 裏返し済みの石の再タップ → 何も起きない(ミスにしない)
4. `flipping` での OK ボタン:
   - 裏返し漏れあり → メッセージ「まだ　ひっくりかえる　こまが　あるよ」、はやおしミス +1
   - 全部裏返し済み → `result` へ
5. `result`: ミス3種の合計が 0 なら `だいせいこう`、それ以外は `せいこう`。「つぎへ」で次の問題の `placing` へ。
6. メッセージ文言は設計書 §2 の表記をそのまま使う(ひらがな・分かち書き)。

### 画面(PracticeScreen)

7. TurnBadge(「あなたは　くろ　です」)・Board・MessageBar・OK/つぎへボタンを配置。OK ボタンは `flipping` 中のみ、「つぎへ」は `result` のみ表示(または活性化)。
8. `flipping` 中は置いた石を強調石(T003 の機能)として表示。「ひっくりかえそう」を MessageBar に表示。
9. `result` では成功メッセージとミス内訳(おくミス・ひっくりかえしミス・はやおしミス、ひらがなラベル)を表示。だいせいこう時は視覚的に特別感(簡易でよい: 大きな花丸文字・色違い等。凝った演出は将来)。
10. プレイ中(placing/flipping)はミスカウントを画面に出さない(設計書 §6)。
11. 仮の問題データ: 開始局面を含む 4〜6 局面をハードコード(1方向1枚だけ返る局面、2方向同時、端・隅を含む局面等バリエーションを持たせ、黒番・白番両方入れる)。最後まで行ったら最初に戻ってループ。

### テスト

12. 状態機械のユニットテスト: 上記 2〜5 の全分岐(合法着手、非合法タップ、正しい石タップ、間違い石タップ、裏返し済み再タップ、漏れあり OK、完了 OK、だいせいこう判定、つぎへ)。盤面は T002 の 64 文字形式で記述し、`flippedDiscs` との整合を検証。
