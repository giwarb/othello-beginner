# T003 最終レビューレポート

## (a) 重大（done を止めるブロッカー）

1. 連続操作すると、石の裏返しアニメーション状態が永久に残ります。

   [Board.tsx](C:/Users/yoshi/work/othello-beginner/app/src/components/Board.tsx:50) では、色が変わった位置を `flipping` に追加し、300ms 後に削除しています。しかし盤面が300ms以内に再更新されると、effect の cleanup が先のタイマーをキャンセルします。

   例えば位置Aを反転し、300ms以内に位置Bを反転すると、Aを削除するタイマーがキャンセルされ、BのタイマーはBしか削除しません。その結果Aは `flipping` に残り続けます。後の手でAが再び反転しても CSS クラスが付いたままなので、アニメーションは再起動しません。

   本要件は「アニメーション完了を待たずに次のタップを受け付けてよい」と連続操作を明示的に許容しています。T004の「石を1枚ずつタップして裏返す」操作で現実に発生するため、要件2を満たすには修正が必要です。

2. デモの表示文言が「すべてひらがな」という要件に反しています。

   [app.tsx](C:/Users/yoshi/work/othello-beginner/app/src/app.tsx:15) の仮メッセージは `${pos}ばんめの ますを おしたよ` で、画面に `45ばんめの…` のような算用数字を表示します。要件6は「文言はすべてひらがな・分かち書き（『OK』のみ例外）」としており、数字は例外に含まれていません。

   幼児向けで文字への依存を減らす方針にも関係するため、ひらがなのみの仮メッセージにするか、仕様側で数字を明示的な例外にする必要があります。

## (b) 中（次タスクで対応すべき）

1. ダークモード環境で主要テキストが読めなくなる可能性があります。

   [index.css](C:/Users/yoshi/work/othello-beginner/app/src/index.css:5) は `color-scheme: light dark` を維持しながら、body の背景を明るい `#eaf5ec` に固定しています。一方、見出しと手番表示の文字色は指定されていません。ダークカラースキームで既定文字色が明色になるブラウザでは、明るい背景や白い手番バッジ上に明色文字が表示され、コントラストが大きく低下します。

   `color-scheme: light` とするか、背景に対応した文字色を明示する必要があります。

   追加されたテストは盤面差分検出などの純関数を確認していますが、effect のタイマー管理やCSSクラスの再付与は対象外です。今回の重大指摘は純関数テストがすべて通っても検出されません。Testing Library導入後、fake timerを使った「300ms以内に複数回盤面を更新する」回帰テストを追加するのが妥当です。
