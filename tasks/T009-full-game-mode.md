## 要件

### 対局の状態機械(UI から独立した純粋ロジック)

1. プレイヤーはくろ(先手)固定、CPU はしろ。開始局面から。
2. **自分の手番**: モードA と同一の placing→flipping フロー(非合法タップ「そこには　おけないよ！」+おくミス、間違い石「そのこまは　ひっくりかえせないよ！」+ひっくりかえしミス、漏れOK「まだ　ひっくりかえる　こまが　あるよ」+はやおしミス)。OK で確定したら CPU 手番へ。ミス3種は**1局を通して累積**。
3. **CPU の手番**: 「あいての　ばん」を表示。CPU は合法手から一様ランダムに選び、置いた石と裏返しは**自動で適用**(裏返しアニメーションは Board の既存機能で発火する)。CPU の思考待ちとして 600〜1000ms 程度の遅延を入れる(即打ちだと何が起きたか分からないため)。
4. **パス**: 自分に合法手がない → 「おける　ばしょが　ないよ！　ぱす！」を表示し、OK ボタンで CPU 手番へ。CPU に合法手がない → 「あいては　ぱす！」を表示して自分の手番へ(自動で進んでよいが、メッセージは読める時間を確保)。
5. **終局**(`isGameOver`): 石数を数えて「くろ　◯こ　しろ　◯こ」を表示し、勝敗メッセージ: プレイヤー勝ち「あなたの　かち！」(だいせいこう相当の花丸演出)、負け「あなたの　まけ」、引き分け「ひきわけ」。ミス3種の内訳も表示する。
6. 終局時に結果(勝敗・石数・ミス合計・完走フラグ)を呼び出し側に通知する仕組みを用意(T010 が購読する。このタスクでは console 出力や callback プロップまでで、保存はしない)。
7. 「もういちど」ボタンで新しい1局を開始できる。「おわる」でホームへ(途中終了は結果通知しない)。

### 画面

8. 練習画面と同じ部品(TurnBadge/Board/MessageBar)を使う。自分の手番では従来どおり「あなたは　くろ　です」、CPU の手番では手番表示が「あいての　ばん」等と分かるようにする。
9. 全文言ひらがな・分かち書き(「OK」のみ例外。石数の数字は許容)。

### テスト

10. gameMachine のユニットテスト: 自分の着手→CPU 応手の遷移、プレイヤーパス、CPU パス、連続パスによる終局、盤面満杯による終局、勝ち/負け/引き分け判定、ミス累積、乱数注入した CPU の着手が合法手であること。
11. 既存テストを壊さない。
