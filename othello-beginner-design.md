# othello-beginner 設計書

作成: 2026-07-15(ユーザー要件ヒアリングに基づきオーケストレーターが言語化)

## 1. コンセプト

**幼児が「実際のオセロ盤で1局打ち切れる」ようになるための練習アプリ。**

普通のオセロアプリと違い、**ひっくり返しを自動でやらない**。実物の盤では自分の手で挟んだ石を全部探して裏返す必要があるため、アプリでも「どこに置けるか」「どの石が返るか」を自分の指で操作して覚える。強さ(戦略の高度さ)は目的ではない。

対象: 幼児(未就学児)。したがって:
- 表示テキストは**すべてひらがな**(分かち書き。カタカナ・漢字・英語を使わない。「OK」ボタンのみ例外)
- 操作は**タップのみ**(ドラッグ・長押し・スワイプなし)
- タップ領域は大きく、誤タップに罰を与えない(メッセージとカウントのみ)
- フィードバックは即時・明るく(成功時は花丸などの演出)

## 2. 練習の基本フロー(1問)

すべての練習問題は同じ2段階フローで進む:

```
局面提示 →「あなたは　くろ　です」
  ↓
【おく段階】置ける場所をタップ
  ↓ (石が置かれる)
【ひっくりかえす段階】「ひっくりかえそう」→ 挟んだ石を全部タップ → OK ボタン
  ↓
結果表示「せいこう！」/「だいせいこう！」+ ミスタッチ内訳
  ↓
つぎのもんだいへ
```

### 2.1 おく段階

- 画面上部に「あなたは　くろ　です」(白番なら「しろ」)と手番の石アイコンを表示
- **合法手をタップ** → 石が置かれ、ひっくりかえす段階へ
- **置けない場所をタップ**(非合法の空きマス・既に石があるマス) → 「そこには　おけないよ！」と表示し、**おくミス +1**。局面はそのまま
- 合法手のガイド表示(候補マスのハイライト)は既定オフ。設定で「ヒントあり」に切替可(導入期用)

### 2.2 ひっくりかえす段階

- 「ひっくりかえそう」と表示。置いた石は目立たせる(枠・光彩)
- **挟まれている相手の石をタップ** → その1枚が裏返る(裏返しアニメーション)
- **挟まれていない石をタップ**(裏返し対象でない相手石・自分の石) → 「そのこまは　ひっくりかえせないよ！」と表示し、**ひっくりかえしミス +1**
- 一度正しく裏返した石は再タップしても戻らない(確定)
- **OK ボタン**を押すと判定:
  - 裏返し漏れがある → 「まだ　ひっくりかえる　こまが　あるよ」と表示し、**はやおしミス +1**。段階続行
  - 全部裏返せている → 結果表示へ

### 2.3 結果表示

- 「せいこう！」と表示し、ミスタッチ内訳を表示:
  - おくミス(置く場所のミスタッチ数)
  - ひっくりかえしミス(裏返しのミスタッチ数)
  - はやおしミス(裏返し漏れのまま OK を押した回数)
- **3種すべて 0 回なら「だいせいこう！」**(特別な演出: 花丸・ファンファーレ等)
- 「つぎへ」ボタンで次の問題へ

## 3. モード構成

### モード A: おいて　ひっくりかえす(ルール練習)

任意の局面(序盤〜終盤のさまざまな局面)が出題され、基本フロー(§2)を1問ずつこなす。正解の「良い手」は問わない — **合法手ならどこに置いてもよい**。ルールの体得(置ける場所の理解と裏返しの完遂)だけが目的。

- 出題局面は実戦的な局面プールからランダム(裏返し枚数・方向数のバリエーションを担保: 1方向1枚だけの局面〜複数方向まとめて返る局面)
- 段階的な難度: さいしょは「1方向だけ返る局面」から、慣れたら「複数方向」へ

### モード B: いいてを　みつける(考える練習)

**誰が見ても明らかに正解が分かる**局面だけを使った、初心者向けの手筋練習。お題がひらがなで表示され、正解の手を置く(置いた後の裏返し段階は §2.2 と同じ)。

カテゴリ(初期セット):

| カテゴリ | お題表示 | 正解の定義 |
|---|---|---|
| すみ | 「すみを　とろう」 | 隅が合法手にある局面。正解 = 隅 |
| Xよけ | 「バツの　ばしょは　やめよう」 | X打ち(隅の斜め隣)が悪手として明白な局面。正解 = X 以外の合法手(C 打ちは扱わない: 幼児には複雑すぎるため、正解にも不正解の説明にも C を持ち込まない。2026-07-15 ユーザー指示) |
| すくなく | 「あいてが　うてる　ばしょが　すくなくなる　ところに　うとう」 | 相手の着手可能数が明確に最少になる手(唯一最少・差が明確)。正解 = その手 |

- **不正解をタップ**したとき: 「ちがうよ！　もういちど」+ お題に応じたヒント表示(例: すみカテゴリなら隅を指し示す)。ミスとしてカウント
- 正解が複数ある局面(Xよけ等)は、どれを選んでも正解
- **問題は各カテゴリ数十問以上を量産する**。生成スクリプト(実戦局面からカテゴリ条件+「明白さ」の閾値でフィルタ)で候補を作り、条件の機械検証を通ったものだけを収録する
- 「明白さ」の基準(生成フィルタの要件):
  - すみ: 隅が打てる。かつ他の手と比べて迷い要素がない(隅を取って直後に取り返される局面は除外)
  - Xよけ: X に打つと直後に相手に隅を取られることが機械的に確認できる。X 以外の安全な代替手が存在する(代替手の良し悪しは問わない)
  - すくなく: 正解手の相手着手可能数が次点より 2 以上少ない(唯一最少)

### モード C: さいごまで　うとう(1局とおしプレイ。2026-07-15 ユーザー指示で実装対象に昇格)

弱い CPU を相手に、1局を最後まで打ち切る。パス・終局・石数勝負を体験するのが目的(実際のボードで1局完走するための総仕上げ)。

- プレイヤーは**くろ(先手)固定**(色選択は将来)。CPU はしろ
- **自分の手番**: 練習モードと同じ(置く→手動ひっくり返し→OK)。メッセージ・ミスカウント3種も同じ挙動で、1局を通して累積する
- **CPU の手番**: 「あいての　ばん」と表示し、少し間を置いて CPU が自動で置き、**自動で裏返る**(実際のボードでも相手の石は相手が返すので、ここは自動でよい)。CPU は弱くてよい: 合法手から一様ランダムに選ぶ
- **パス**: 自分に合法手がないとき「おける　ばしょが　ないよ！　ぱす！」を表示し「OK」で相手番へ。CPU がパスのとき「あいては　ぱす！」を表示
- **終局**(双方に合法手なし or 盤面が埋まった): 石数を数えて表示(例: 「くろ　４０こ　しろ　２４こ」。個数の数字はミス回数と同様に許容)。勝敗メッセージ: 勝ち「あなたの　かち！」(花丸演出)、負け「あなたの　まけ」、引き分け「ひきわけ」
- 「おわる」でいつでもホームへ戻れる(途中終了は記録しない)
- ホームに「さいごまで　うとう」ボタンを追加

### がんばりの記録(スタンプちょう。2026-07-15 ユーザー指示で実装対象に昇格)

- localStorage に累積保存: (1) 練習の「だいせいこう」回数、(2) 1局完走回数、(3) かち回数
- ホームに「すたんぷ」ボタン → 記録ページ: だいせいこう1回につきスタンプ(◎や⭐等)を1つ並べて表示。完走・かちも同様のスタンプ行で表示。数字表記より「スタンプが並んで増える」見た目を優先(幼児が達成を実感できる)
- 表示上限(例: 1行10個で折り返し、多すぎる場合は「+まとめ表示」)は実装者裁量。リセット機能は付けない

### UI 共通の追加方針(2026-07-15 ユーザー報告起点)

- **テキスト選択・キャレットの禁止**: 盤面・メッセージ・ボタン等のアプリ UI テキストで、タップ/クリックによりテキスト選択やキャレット(文字カーソル)が出てはならない(`user-select: none` 等)。幼児が連打してもテキスト選択状態にならないこと

## 4. 技術構成

othello-trainer の実績構成を簡素化して踏襲する。**エンジン(Rust/WASM)は使わない** — 強さ不要のため、ロジックはすべて TypeScript。

- **UI**: Preact + TypeScript + Vite(`/app` ディレクトリ)
- **盤面**: SVG 描画(Canvas より実装が簡素。8x8 でパフォーマンス問題なし)。裏返しアニメーションは CSS/SVG transform
- **コアロジック** (`app/src/core/`): 8x8 配列ベース。合法手判定・挟まれる石の列挙・着手適用・パス/終局判定。Vitest でユニットテスト
- **問題データ**: 静的 JSON(`app/src/data/` または `app/public/`)。形式は「盤面(64文字列等)+手番+カテゴリ+正解手リスト+メタ」
- **問題生成**: `/puzzlegen`(TypeScript スクリプト、Node で実行)。ランダム自己対戦で局面を量産し、§3 の機械検証フィルタで絞る。生成物の JSON はコミット対象
- **配信**: GitHub Pages(GitHub Actions でビルド・デプロイ)。オフライン対応(PWA)は将来
- **テスト**: Vitest(コアロジック・判定ロジック・問題データの妥当性検証)

## 5. データ形式(初期案)

```ts
// 局面: 64文字。'-'=空, 'b'=黒, 'w'=白。a1(左上)から行優先
interface Puzzle {
  id: string;
  mode: 'rule' | 'strategy';
  category?: 'corner' | 'avoid-x' | 'min-mobility'; // strategyのみ
  board: string;        // 64文字
  turn: 'b' | 'w';
  answers?: number[];   // strategyのみ: 正解マス(0-63)。ruleでは全合法手が正解
  difficulty: 1 | 2 | 3; // rule: 1=1方向1枚 〜 3=複数方向
}
```

問題データの妥当性(盤面が到達可能か、answers が本当に合法手か、カテゴリ条件を満たすか)は、生成フィルタと同じ検証関数をテストでも回して担保する。

## 6. 画面構成(初期案)

1. **ホーム**: モード A / モード B(カテゴリ選択) の大きなボタン
2. **練習画面**: 手番表示、盤面、メッセージ帯、OK/つぎへボタン。ミスカウントは内部で保持し結果画面でのみ表示(プレイ中に数字を出して萎縮させない)
3. **結果画面**: せいこう/だいせいこう + ミス内訳 + つぎへ
